unl-maps-sdk-ios

Introduction

unl-maps-sdk-ios is a mapping SDK for developers of iOS applications, extending and enhancing functionalities of MapLibre GL iOS. Apart from the capabilities of maplibre, the package exposes the following controls:

  • Grid lines with a customisable precision;

  • Tile selector, including ‘vectorial’, ‘traffic’, ‘terrain’, ‘satellite’ and ‘base’ options;

Getting started

Installing

Add package dependency by following below steps

  • Go to the project -> file -> add Package

  • Search unl-maps-sdk-ios

  • Use “main” branch

  • Click Add packege

Usage in your application

Swift

When using SDK

import UNLMapSDK

Initializing the SDK

Example usage

AppDelegate.swift


    /// Initializing the UNL SDK
    let unlSDK = UNLMapSDK()

    func application(_ application: UIApplication, didFinishLaunchingWithOptions launchOptions: [UIApplication.LaunchOptionsKey: Any]?) -> Bool {
        /// Override point for customization after application launch.

        /// Configure UNL SDK while launching the App
        /// apiKey: and vpmId is mandatory to configure the sdk
        /// Set gridControl: flag true to display control, By defaults its false
        /// Set tilesSelectorControl: flag true to display inbuilt control, By defaults its false 
        unlSDK.configure(apiKey: "<YOUR-UNL-API-KEY>", vpmId: "<YOUR-VPM-ID>", gridControl: true, tilesSelectorControl: true)

        return true
    }

See SDK Authentication Docs for instructions on how to generate an apiKey and vpmId to get started with the UnlSdk.

Custom controls

For Custom controls, it can be accessed within any UIViewController by assigning a class UNLMapView to UIView and by creating it’s outlet. For Example:

    @IBOutlet var mapView: UNLMapView!

Grid Controls

Grid controls can be enabled during the AppDelegate.swift initialisation by passing the gridControl option true. With this approach, the grid control will be initialised with the default options.

Grid control default options can also be changed after the initialisation by calling the gridOptions function on the map reference:

        self.mapView.gridOptions = GridControlOptions(.GEOHASH_LENGTH_9, .red, 0.5, .systemYellow, .systemYellow)

the following options can be specified during the GridControl initialisation:

Option Type Default Description
defaultPrecision CellPrecision 9 Default precision of the cells. It can be changed manually via the grid selector
lineColor UIColor .lightGray Grid line’s colour
lineWidth Double 0.5 Grid line’s width
cellFillColor UIColor .systemYellow Cell Fill Color
cellBorderColor UIColor .systemYellow Cell Border Color

If GridControl is enabled, the grid lines will be generated at certain zoom levels, dependant on the selected precision, according to the following table:

Precision Zoom level
10 20
9 18
8 16
7 14
6 12
5 10
4 8
3 4
2 3
1 2

Tiles selector

Tile selector control can be enabled during the AppDelegate.swift initialisation by passing the tilesSelectorControl option true. With this approach, the tile selector control will be initialised with the default options.

the following options can be specified during the TilesSelectorControl initialisation:

self.mapView.arrayTiles = [.base, .satellite, .terrain, .traffic, .vectorial]
Option Type Default Description
tiles [Tiles] [“vectorial”, “traffic”, “terrain”, “satellite”, “base”] The options that will be included in the tiles selector

After setting up all custom controls, apply them over the map by following:

        /// Load map after setting properties
        self.mapView.loadMap()

Custom tile selector

UNLMapView also provide access to use custom tile selecter with the following function:

func changeTile(style:Tiles)

This method takes a Tiles parameter and updates the selected tile from the map.

The supported Tiles values are: vector, traffic, terrain, satellite, base.

Example


mapView.changeTile(style: .terrain)
